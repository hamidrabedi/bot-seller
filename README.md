# bot-seller (Django + Telegram + 3x-ui)

Production-oriented minimal stack:
- Django admin
- Django REST API
- Telegram bot (FA/EN)
- 3x-ui provisioning only
- Manual bank transfer + receipt screenshot upload (temporary payment flow)
- SQLite

## 1) Fast install (interactive)
```bash
chmod +x install.sh
./install.sh
```
The script will:
- ask for env values,
- create `.env`,
- create virtualenv,
- install dependencies,
- run migrations.

## 2) Start services
### API
```bash
source .venv/bin/activate
python manage.py runserver 0.0.0.0:8000
```

### Telegram bot
```bash
source .venv/bin/activate
python manage.py run_telegram_bot
```

## 3) Admin setup
```bash
source .venv/bin/activate
python manage.py createsuperuser
```
In `/admin` add:
- one **Panel3XUI** (url, username, password, inbound_id)
- one or more **Plan**
- one **PaymentSettings** with your own custom bank card text (FA/EN)

## 4) Payment flow (current)
1. User asks for bank transfer info.
2. User transfers money manually.
3. User uploads screenshot receipt.
4. Admin reviews `PaymentReceipt` in admin and sets status to `approved`.
5. Then service creation API can issue config.

## 5) API endpoints
- `GET /api/health/`
- `GET /api/plans/`
- `GET /api/payment/bank-info/`
- `POST /api/payment/upload-receipt/` multipart with `telegram_user_id`, `plan_id`, `screenshot`
- `POST /api/services/create/` body: `telegram_user_id`, `plan_id` (requires approved receipt)
- `GET /api/services/my/?telegram_user_id=123`

## 6) Telegram receipt format
- Send receipt image with caption: `receipt:PLAN_ID`
- Example: `receipt:1`

## 7) Environment file
Generated by installer (`.env`):
- `DJANGO_SECRET_KEY`
- `DEBUG`
- `ALLOWED_HOSTS`
- `TELEGRAM_BOT_TOKEN`
